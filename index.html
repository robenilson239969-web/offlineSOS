OfflineSOS: Comunicação offline entre usuários em situações de emergências ou isolamento.

Introdução

LoRa é uma tecnologia de rádio de baixa largura de banda que permite comunicações longas com baixo consumo de energia — ideal para sensores remotos e aplicações IoT. Neste guia mostramos como conectar um módulo RFM95 a um ESP32 e como programar um transmissor e um receptor usando a Arduino IDE.

Esta técnica de modulação permite a comunicação de longo alcance de pequenas quantidades de dados (o que significa baixa largura de banda), alta imunidade a interferências e minimiza o consumo de energia. Assim, permite comunicação de longa distância com baixos requisitos de energia.

Componentes Utilizados

  • 2x ESP32 DOIT DEVKIT V1
  • 2x Módulo LoRa RFM95 (adequado para a frequência da sua região)
  • Cabos jumper, breadboard ou stripboard
  • Antena apropriada para o módulo (importante!)
  • Computador com Arduino IDE configurado para ESP32

Frequências e antena

As bandas LoRa mais comuns são 868 MHz (Europa), 915 MHz (América do Norte) e 433 MHz (Ásia). Verifique as regras locais antes de operar e ajuste a antena ao comprimento correto para a frequência escolhida.

Exemplo de código (Sender)

Abaixo, um sketch de exemplo (simplificado) que transmite uma mensagem "hello" com um contador. Use a biblioteca LoRa de Sandeep Mistry.

// Sender (ESP32) - exemplo simplificado
#include <SPI.h>
#include <LoRa.h>

#define ss 5
#define rst 14
#define dio0 2

int counter = 0;

void setup() {
  Serial.begin(115200);
  while(!Serial);
  Serial.println("LoRa Sender");
  LoRa.setPins(ss, rst, dio0);
  if (!LoRa.begin(868E6)) { // ajuste para sua frequência
    Serial.println("Falha ao inicializar LoRa");
    while (1) { delay(1000); }
  }
}

void loop() {
  LoRa.beginPacket();
  LoRa.print("hello ");
  LoRa.print(counter++);
  LoRa.endPacket();
  delay(10000); // enviar a cada 10s
}

Substitua a frequência (868E6, 915E6, etc.) conforme a sua região.

Exemplo de código (Receiver)

O receptor inicializa o rádio na mesma frequência e imprime mensagens recebidas no Serial Monitor. A lógica é parecida com o exemplo do Sender — apenas utilize LoRa.parsePacket() e LoRa.readString() para ler os dados.

Dicas práticas

  • Sempre use antena; sem ela o alcance cai drasticamente.
  • Verifique a alimentação: forneça corrente estável a 3.3V.
  • Se usar módulos RFM95 sem breakout, considere soldar fios ou usar um breakout compatível.
  • Teste a comunicação com o Arduino Serial Monitor para diagnosticar problemas.
